Short spec

Data model foundation

Generalize today's fixed array into nestable pieces:

  • Step — degree · joystick dir · gate · rest (unchanged)
  • Pattern — [Step] of N bars (1 → 2 → 4), paged one bar at a time
  • Lane — a Pattern + sound + mute/solo + loopLen; loops on the MasterClock
  • SavedPattern — a named Pattern in the Library, loadable into a lane

Persistence: seqSteps.v1 → v2 (lanes); Library stored separately.

Phase 1 — fix today's screen now

  • spaceFill the wasted margins; sequencer owns full height, both columns.
  • editorRoomy step editor in the freed left half — usable while playing, with a small ↶ undo.
  • gateMake Gate audible: <30% staccato, ~100% legato / tie. The note-bar previews it.
  • cornerTransport (Play) anchored bottom-right corner.

Phase 2 — bars & pagination (B) next

  • bars1 / 2 / 4 bars; bar tabs 1·2·3 + corner ＋, all at the bottom.
  • playheadMini-map sweeps the whole pattern; view auto-flips bars as it crosses (Play).
  • chain⟳ chain plays all bars in sequence vs. looping the current bar.

Phase 3 — lanes + library (A) the core ask

  • buildStart with one empty lane; add more over time. No prefilled clutter.
  • layerStack lanes, each on its own length; M/S to arrange live.
  • librarySave any lane → reusable pattern in the Library; load it into a new lane or fire it alongside live play.

Deferred: "record from live play" and full song-block arrangement — revisit once the Library fills up.
